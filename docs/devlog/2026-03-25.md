## 下午继续：Railway 云端部署

**架构决策**
- 开源自部署模式：每人 clone 代码、部署自己的 Railway 实例，数据完全隔离
- SaaS 改造留到 Phase 3/4：需要登录系统、权限校验、Strava Extended Access 审核
- 数据库切换：SQLite（本地）→ PostgreSQL（Railway 免费提供）

**代码改动**
- `requirements.txt`：加 `psycopg2-binary`
- `config.py`：新增 `sqlalchemy_database_url` 属性，自动将 Railway 的 `postgresql://` 转成 `postgresql+psycopg2://`
- `database.py`：兼容 SQLite 和 PostgreSQL，SQLite 专用的 `check_same_thread` 和 PRAGMA 迁移逻辑仅在 SQLite 模式下执行
- `backend/Procfile`：告诉 Railway 用 `uvicorn main:app --host 0.0.0.0 --port $PORT` 启动
- `backend/nixpacks.toml`：显式声明构建和启动命令，解决 Railpack 无法识别项目类型的问题

**Railway 部署过程**
- 新建项目 → 连接 GitHub → 添加 PostgreSQL 服务
- 后端服务：Root Directory = `backend`，Variables 配置 Strava / Anthropic / DB 环境变量
- 前端服务：Root Directory = `frontend`，`NEXT_PUBLIC_API_URL` 指向后端公网地址
- 踩坑：Root Directory 不能带 `/`（`/backend` 要改成 `backend`）
- 踩坑：前端端口实际是 8080，不是 3000（Railway 自动分配 PORT，删掉手动设置的 PORT=3000）
- 踩坑：同步接口是 POST，浏览器只能 GET，改成 GET 解决
