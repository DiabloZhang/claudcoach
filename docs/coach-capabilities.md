# Coach 能力清单

> 随产品迭代持续更新。标注实现阶段：[3.x] = Phase 3，[4.x] = Phase 4

---

## 一、训练复盘（核心）[3.2]

活动同步后主动发起，通过对话收集：
- 训练类型（间歇 / 节奏 / 有氧 / 恢复 / 长距离）
- 主观感受 RPE（1-10）
- 身体状态（正常 / 疲劳 / 疼痛 / 生病）
- 生活干扰（工作 / 睡眠 / 情绪等）
- 用户补充内容（"学员说，xxx"写入日志）

---

## 二、训练记录与记忆 [3.2]

### 每日日志（sleep_time 触发 + 新活动/对话后更新）
- 单字段 `content`，教练统一写
- 同时作为入睡提醒

### 周总结（周日 sleep_time 触发）
### 月总结（月末 sleep_time 触发）
### 赛季总结（赛季结束后触发）
### 比赛复盘（比赛后触发）

所有记录可在「日志」页面查看（Feed 流 + 日历视图）

---

## 三、赛事管理 [3.3]

### 赛事信息页
- 独立页面，展示本赛季所有比赛
- 字段：赛事名、日期、距离类型（Sprint/Olympic/70.3/113/FullIM）、地点、目标成绩

### 比赛识别
- 主动告知：用户对话中提到 → 教练确认 → 写入
- 事后猜测：活动名含"Race/比赛/赛"、周末、同日三项、距离吻合 → 教练主动问

### 赛前服务（比赛前 7 天进入赛前模式）
- 训练减量提醒（tapering）
- 行程规划建议（出发时间、提前几天到达）
- 住宿建议（距离赛场、安静区域）
- 赛前一天安排提醒（注册领包、器材检查、饮食、睡眠）
- 赛前热身建议

### 赛后复盘
- 成绩录入（对话中说，教练记）
- 各项分段分析
- 与目标对比
- 下一个赛事备战建议

---

## 四、目标与赛季规划 [3.3]

- 赛季目标通过对话建立，不填表
- 无目标也是合法状态，教练定期问
- 根据赛事日期倒推训练阶段（基础期 / 积累期 / 专项期 / 减量期）
- TSB 过低时主动预警

---

## 五、主动触发场景 [3.3]

| 触发条件 | 教练行为 |
|---------|---------|
| 新活动同步 | 发起训练复盘对话 |
| 每天 sleep_time | 写日志 + 入睡提醒 |
| 周日 sleep_time | 周总结 |
| 月末 sleep_time | 月总结 |
| 比赛前 7 天 | 进入赛前模式 |
| 比赛后 1 天 | 发起赛后复盘 |
| TSB < -30 | 疲劳预警 |
| 连续 3 天未训练 | 关怀询问 |
| 目标未设置 | 每次对话自然问 |

---

## 六、教练任务队列（CoachTask / ❓）[3.2]

头顶 ❓ 显示未解决任务，进入对话后主动处理：
- `goal_missing`：目标未设置
- `injury_followup`：上次提到伤痛，未追踪
- `race_debrief`：比赛结束未复盘
- `plan_check`：周计划完成情况未确认

---

## 七、教练人设与个性 [3.1]

- 名字、性格、背景由用户自定义
- 心情值（0-100）影响语气：🤩 亢奋 / 🙂 温和 / 🙃 冷静 / 😤 冷淡
- sleep_time 和 timezone 教练在对话中询问，不填表

---

## 八、语音交互 [3.4]

- STT / TTS：字节火山引擎
- PC 默认打字，手机默认语音（参考豆包交互模式）
- 声音克隆（模仿运动明星等）：[4.x] nice-to-have
